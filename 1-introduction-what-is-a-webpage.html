Webページって何？ - デジタル世界の入り口を理解しよう

インターネット上に表示される文書やコンテンツのことをWebページと呼びます。 Webページ（Web page）: インターネットブラウザで閲覧できる文書やコンテンツのことです。文字、画像、動画などを含み、HTMLという言語で書かれています。「Web」は「蜘蛛の巣」を意味し、ページ同士がリンクで繋がっている様子を表しています。

👥 今日の始まり

太郎: あきらパパ、今度はWebページの作り方を教えてもらえるんですよね？
あきらパパ: はい！前回のGitで基本的なファイル管理はできるようになったので、今度は実際にWebページを作ってみましょうか。
太郎: わあ、ついに！でも、そもそもWebページって何なんですか？普段見てるサイトがどうやって作られてるのか、全然想像がつかなくて...
あきらパパ: とてもいい質問ですね。実は、Webページって太郎くんが毎日書いてる日記帳みたいなものなんですよ。
太郎: えっ、日記帳ですか？

📔 Webページって実は「デジタルの日記帳」なんです

日記帳で考えてみよう

あきらパパ: 太郎くんは学校で作文を書いたり、日記を書いたりしますよね？
太郎: はい！夏休みの日記とか、読書感想文とか。
あきらパパ: その日記や作文には、どんなものが入ってますか？
太郎: えーっと...タイトルがあって、日付があって、本文があって...あ、時々絵も描きます！
あきらパパ: そう！それがまさにWebページなんです。

あきらパパ: そうなんです！だから、文章が書ける人なら、誰でもWebページが作れるんですよ。

✏️ 作文とHTMLの関係

作文の書き方ルールを思い出してみよう

あきらパパ: 太郎くんが作文を書く時、どんなルールがありますか？
太郎: えーっと...タイトルは大きく書いて、段落を分けて、大事なところは線を引いたり...
あきらパパ: 素晴らしい！それがまさにHTMLの考え方なんです。 HTML（HyperText Markup Language）: Webページを作るための基本的な言語です。「マークアップ言語」と呼ばれ、テキストに「これは見出し」「これは段落」などの印（マーク）をつけていきます。「ハイパーテキスト」とは、リンクを通じて別のテキストにジャンプできる仕組みのことです。

📝 作文のルール → HTMLのルール

作文 HTML 意味
太い字で大きくタイトル <h1>タイトル</h1> 大見出し（heading 1の略）
段落を分ける <p>文章</p> 段落（paragraphの略）
大事な部分に線を引く <strong>大事</strong> 強調（強い強調を意味する）
写真を貼る <img src="写真.jpg"> 画像（imageの略、srcは出典元を指定）

実際にHTMLを見てみよう！

太郎: HTMLってどんな感じなんですか？
あきらパパ: 実は、とってもシンプルなんです。太郎くんの日記をHTMLで書くとこんな感じ：
<h1>夏祭りに行ったよ</h1> <p>今日は家族で夏祭りに行きました。</p> <p>綿あめが美味しかったです。</p> <img src="お祭りの写真.jpg">
太郎: わあ！意外と普通の文章みたいですね！
あきらパパ: そうでしょう？タグとか要素とかは、「ここは見出しですよ」「ここは段落ですよ」って印を付けてるだけなんです。 タグ（Tag）: HTMLで使われる<h1>や<p>などの記号のことです。「ここから」と「ここまで」を示すために、開始タグ（<p>）と終了タグ（</p>）がペアになっています。「タグ付け」することで、ブラウザにその部分がどのような役割を持つか伝えます。 要素（Element）: 開始タグから終了タグまでの全体を指します。例えば、<p>これは段落です</p>という全体が「p要素」です。要素の中に別の要素を入れることもできます（入れ子構造）。

太郎: 年賀状ですか？
あきらパパ: そう！こんな流れですよね：

太郎: なるほど！郵便屋さんがインターネットで、ポストがサーバーなんですね！ サーバー（Server）: インターネット上でWebページのファイルを保存し、世界中のユーザーにデータを提供するコンピューターのことです。「給仕する人」という意味で、リクエストに応じてデータを「提供する」役割を持っています。24時間365日、常に稼働していて、誰かがWebページを見たいと思ったときにすぐに提供できる状態を維持しています。

📱 どんな機械でも見れる不思議

同じ手紙、違う読み方

太郎: あ、そういえばスマホでもパソコンでも同じサイトが見れますよね？でも、画面の大きさが全然違うのに、どうして同じように見れるんですか？
あきらパパ: 太郎くん、とても良い気づきですね！実は、これも手紙に例えるとわかりやすいんです。

📜 同じ手紙を、読む人によって見え方が変わる

flowchart TD A[同じHTMLコンテンツ] --> B[パソコンで見る] A --> C[スマホで見る] A --> D[タブレットで見る] B --> E[大きな画面表示] C --> F[小さな画面表示] D --> G[中くらいの画面表示] style A fill:#4CAF50,stroke:#388E3C,color:#fff style B fill:#9575CD,stroke:#7E57C2,color:#fff style C fill:#9575CD,stroke:#7E57C2,color:#fff style D fill:#9575CD,stroke:#7E57C2,color:#fff

  • 🔍 お母さん：普通の大きさで読む
  • 😎 太郎くん：好きな明るさで読む

→ 手紙の内容（HTML）は同じだけど、読み方（見え方・CSS）が違う！

太郎: ああ！中身は同じで、見せ方が変わるんですね！でも、どうやって自動で調整されるんですか？
あきらパパ: その通り！HTMLは中身（日記の内容）で、見た目は別の方法で決めるんです。それを次回のCSSで教えますね。 CSS（Cascading Style Sheets）: HTMLで作った構造に対して、色やサイズ、配置などの見た目（スタイル）を指定するための言語です。HTMLが「何を表示するか」を決めるのに対し、CSSは「どのように表示するか」を決めます。「Cascading（階層的）」とは、複数のスタイル指定がある場合に、どのスタイルを優先するかを決める仕組みのことです。
太郎: うーん、でもまだちょっとピンとこないです...実際にはどうなってるんですか？
あきらパパ: じゃあ、もっと身近な例で説明しましょう。太郎くん、電子書籍って使ったことありますか？
太郎: あ、はい！タブレットで漫画を読んだことがあります！
あきらパパ: そう！電子書籍も同じなんです。同じ漫画でも、タブレットを縦にしたり横にしたりすると、自動でページの表示が変わりますよね？
太郎: そうか！Webページも同じように、見る機械に合わせて自動で調整されるんですね！すごい！

<h1>大きな見出し</h1> ← タイトル用 <p>普通の文章</p> ← 文章用 <img src="写真.jpg"> ← 写真用 <ul><li>リスト</li></ul> ← 箇条書き用 <strong>大事な言葉</strong> ← 強調用
太郎: たった5つですか？思ったよりぜんぜん少ない！
あきらパパ: そうなんです！最初はこの5つだけで十分なんですよ。
太郎: でも、なんで< >で囲むんですか？普通に書いちゃダメなの？
あきらパパ: いい質問！これは「ここからここまでが見出しですよ」って、コンピューターに教えてあげるための目印なんです。 マークアップ（Markup）: テキストに「印をつける」作業のことです。HTMLでは<h1>などの「タグ」を使って、テキストの役割や構造を示します。コンピューターはこれらの印を見て、「ここは見出しだから大きく表示しよう」「ここは段落だから適切な間隔をあけよう」などと判断します。
太郎: なるほど！コンピューターは賢そうに見えて、実は「ここは見出し」って教えてあげないとわからないんですね！
あきらパパ: その通り！太郎くんの理解力、素晴らしいですね。
